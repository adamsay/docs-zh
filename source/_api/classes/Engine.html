_canModifyCurrentScene Boolean private

Defined in utils/api/engine/engine.js:119

!#en You should check whether you can modify the scene in constructors which may called by the engine while deserializing. !#zh 这个属性用来判断场景物体的构造函数执行时是否可以把物体加到场景里。 这个属性和 Fire._isCloning 很类似。但这里关注的是场景是否能修改，而 Fire._isCloning 强调的是持有的对象是否需要重新创建。

_scene Scene private

Defined in utils/api/engine/engine.js:29

!#en the active scene !#zh 当前激活的场景。

如果为空，一般是因为正在加载场景或Entity(例如执行Fire.deserialize)。 这样是为了防止加载中的东西不小心影响到当前场景。一般代码不用关心这个问题，但大部分构造函数里执行的代码， 如果涉及到场景物件的操作，都要注意这点。 也就是说构造函数调用到的代码如果要操作 Engine._scene，必须判断非空，如果操作不直接针对 Engine._scene， 也可以判断 Engine._canModifyCurrentScene。 另外，如果存在辅助场景，当在辅助场景内创建物件时，Engine._scene会被临时修改为辅助场景。

preloadScene
(
  • sceneName
  • [onLoaded]
)

Defined in utils/api/engine/engine.js:451

Preloads the scene to reduces loading time. You can call this method at any time you want.

After calling this method, you still need to launch the scene by Engine.loadScene because the loading logic will not changed. It will be totally fine to call Engine.loadScene at any time even if the preloading is not yet finished, the scene will be launched after loaded automatically.

Parameters:

  • sceneName String
    • the name of the scene to preload
  • [onLoaded] Function optional
    • callback, will be called after the scene loaded
    • param String

      error - null or the error info

    • param Asset

      data - the loaded scene or null
